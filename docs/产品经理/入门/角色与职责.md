# 产品经理的角色与职责

---

## 产品经理是什么

产品经理（Product Manager，简称 PM）是负责产品全生命周期管理的角色，从需求发现到产品上线迭代，始终以用户价值为导向。

**核心定位**：产品经理是连接用户、技术和业务的桥梁。

---

## 日常职责

### 需求管理

| 环节 | 内容 |
|------|------|
| **需求收集** | 收集来自用户、市场、运营、老板等各方需求 |
| **需求分析** | 辨别真需求与伪需求，评估需求价值 |
| **需求优先级** | 使用 RICE、Kano 等模型确定优先级 |
| **需求文档** | 编写 PRD（产品需求文档） |

### 产品设计

| 环节 | 内容 |
|------|------|
| **原型设计** | 使用 Axure、Figma 等工具制作原型 |
| **交互设计** | 优化用户操作流程和体验 |
| **PRD 编写** | 详细描述功能需求、业务流程 |

### 项目跟进

| 环节 | 内容 |
|------|------|
| **需求评审** | 组织开发、设计、测试进行需求评审 |
| **进度跟踪** | 跟进开发进度，解决开发中的问题 |
| **测试验收** | 参与测试，确保产品功能符合预期 |
| **上线发布** | 协调上线，准备发布文档和培训材料 |

### 数据分析

| 环节 | 内容 |
|------|------|
| **数据埋点** | 设计关键事件埋点方案 |
| **数据分析** | 分析用户行为和产品表现 |
| **优化迭代** | 基于数据提出优化建议 |

---

## 与其他角色的区别

### vs 项目经理

| 对比项 | 产品经理 | 项目经理 |
|--------|---------|---------|
| **核心关注** | 产品价值 | 项目交付 |
| **职责重心** | 定义"做什么" | 确保"怎么做" |
| **成功标准** | 产品是否成功 | 项目是否按时完成 |

### vs 运营

| 对比项 | 产品经理 | 运营 |
|--------|---------|------|
| **关注阶段** | 产品设计和迭代 | 产品推广和用户活跃 |
| **核心能力** | 需求分析、产品设计 | 用户运营、活动策划 |

### vs 开发

| 对比项 | 产品经理 | 开发 |
|--------|---------|------|
| **关注点** | 用户价值和体验 | 技术实现和性能 |
| **决策方式** | 需求驱动 | 技术驱动 |
