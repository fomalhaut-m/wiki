# 🎯 Luke 的产品经理转型学习路径

---

## 📌 第一阶段：产品思维底座（1-2个月）

### 核心知识

| 方向 | 学什么 |
|------|--------|
| **用户需求挖掘** | 需求采集方法、用户访谈、痛点分析 |
| **产品设计** | MVP思维、PRD写作、需求优先级（RICE/Kano模型） |
| **竞品分析** | SWOT、对比表格、用户体验地图 |
| **商业模式** | 商业画布（BMC）、单元经济学、NPS |

### 实践任务

- [ ] 选一个熟悉的领域（AI应用/游戏/电商），做一份**竞品分析报告**
- [ ] 用产品视角重新描述做过的项目，写一份**简化版PRD**

## 📌 第二阶段：AI时代的差异化能力（贯穿全程）

### 这是你最大的竞争力

你的简历里有：AI Agent设计、Function Calling、多AI模型集成、Token成本优化——这是2026年最稀缺的产品能力。

| 能力项 | 学什么 |
|--------|--------|
| **AI产品设计** | 如何设计AI-native功能、人机协作流程、prompt设计 |
| **AI产品思维** | AI边界判断（什么时候用AI vs 不用）、AI伦理合规 |
| **大模型应用** | RAG、Agent架构、多模型选型策略 |
| **AI成本意识** | Token成本模型、模型调用优化（你已经有经验） |

### 实践任务

- [ ] 设计一个**AI辅助的产品功能**，结合做过的AI剧本/翻译系统
- [ ] 分析市面上的AI产品（Cursor、Copilot等），写产品体验报告

## 📌 第五阶段：简历 & 面试转型（最后冲刺）

### 技术简历 → 产品简历 的转化重点

| 技术写法 | 产品写法 |
|---------|---------|
| "15+微服务架构设计" | "设计全球化平台架构，支撑日均百万级交易" |
| "核心接口响应从300ms优化至60ms" | "通过性能优化，TPS提升300%，用户体验显著提升" |
| "AI调用成本降低38%" | "通过AI能力落地，实现内容创作效率提升80%" |

### 面试重点：你的5个核心优势

1. **技术深度**：能跟研发平等对话，避免"需求说不清楚"
2. **架构思维**：不只是功能级产品，能做系统级产品设计
3. **AI落地经验**：做过真实AI产品（剧本生成、多语言翻译、Agent设计）
4. **商业敏感度**：降本38%、节省6000$/月——懂技术和商业的结合
5. **B端/复杂业务经验**：千亿级平台、跨区域多活——硬核经历
