# 产品经理入门基础

## 一、什么是产品？

**定义**：解决了用户问题的东西（有形的产品、无形的产品）

产品可以是实体商品、软件应用、服务或解决方案。

## 二、什么是产品经理？

**定义**：负责产品需求分析、产品功能设计、产品测试、产品发布等

> 产品经理是"想清楚产品怎么做的人"

产品经理的核心职责是将用户需求转化为可实现的产品方案。

## 三、合格产品经理的核心能力

### 1. 问题分析能力

正确理解问题的关键要素：

| 要素 | 说明 |
|------|------|
| **谁** | 用户角色 |
| **什么情况下** | 场景/情境 |
| **遇到什么事情** | 问题/痛点 |
| **什么时间地点环境** | 时空上下文 |

### 2. 产品设计能力

思考：如何设计才能更好地解决用户问题？

设计考量维度：

| 维度 | 说明 |
|------|------|
| **功能** | 解决什么问题 |
| **体验** | 如何让用户用得舒服 |
| **优势** | 相比竞品的差异化 |
| **价值** | 为用户带来的实际价值 |

## 四、产品的开发流程

```
分析定义 → 软件设计 → 编码测试 → 运行维护
```

| 阶段 | 说明 |
|------|------|
| **分析定义** | 问题定义、可行性研究、需求分析 |
| **软件设计** | 概要设计、详细设计 |
| **编码测试** | 程序编码、集成测试 |
| **运行维护** | 部署上线、运行监控、问题处理 |

## 五、产品经理的主要工作

### 1. 评估产品机会

- 需求收集
- 可行性研究
- MRD / 立项工作

### 2. 定义要开发的产品

- 需求分析
- 原型设计
- PRD（产品需求文档）

## 六、岗位职责误区

| 误区 | 说明 |
|------|------|
| **画原型的** | 容易与需求脱节，回避探索定义产品的过程，限制创新思维 |
| **管项目的** | 职责混淆，产品经理专注产品定义和价值，项目经理关注执行交付 |
| **需求混淆** | 老板/用户/自己的需求 ≠ 产品需求，需探寻需求本质及可行性 |
| **职权混淆** | 产品经理对产品生命周期负责，驱动同级协作，无项目管理权 |

| **体验设计** | 用户体验、产品体验、原型设计、交互设计 |
| **文档工具** | BRD、MRD、PRD、立项说明书、产品说明书 |
