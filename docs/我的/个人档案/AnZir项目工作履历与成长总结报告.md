# AnZir项目工作履历与成长总结报告

## 一、项目概览

在AnZir项目中，我参与了一个大型AI驱动的游戏内容创作平台的开发，该项目包含15个以上的微服务，涵盖了AI集成、游戏引擎、电商系统、支付系统、数据分析等多个技术领域。项目采用Spring Boot 3.2.12、Spring Cloud微服务架构，部署在AWS ECS/Fargate云原生环境中。

## 二、承担的主要职责

### 1. AI Agent中心开发
- 负责AI Agent中心的核心模块设计与开发，包括会话管理、计费系统、限流机制等
- 实现了AI翻译、AI剧本创作、AI NPC对话等特色功能
- 开发了19个Function Calling工具，扩展AI能力边界，使其能够处理复杂的业务场景

### 2. AI模型抽象层设计
- 设计并实现多AI模型抽象层，支持OpenAI、DeepSeek等多种AI提供商的无缝切换
- 通过统一的抽象层提升了系统的灵活性和可扩展性

### 3. 性能优化与系统架构
- 优化系统性能，通过合理的资源配置和缓存策略提升系统吞吐量
- 参与微服务架构设计，确保各服务间的松耦合和高内聚
- 实现分布式锁和并发控制机制，解决高并发场景下的数据一致性问题

### 4. 基础设施建设
- 设计并实现标准化的容器镜像，提升部署效率和安全性
- 部署和配置Metabase数据分析平台，为业务决策提供数据支持

## 三、技术成长与收获

- 学会了如何在分布式环境下保证数据一致性和系统稳定性

### 3. 高并发系统处理经验
- 通过Redis限流和异步处理应对高并发AI请求
- 掌握了分布式锁的使用场景和实现方法

## 五、技术挑战与解决方案

### 1. AI模型切换挑战
- **挑战**：不同AI提供商的API差异较大，切换成本高
- **解决方案**：设计统一的AI模型抽象层，实现多提供商的无缝切换

### 2. 高并发AI请求处理
- **挑战**：AI API调用成本高，并发请求可能导致费用失控
- **解决方案**：通过Redis限流和异步处理机制控制请求频率

### 3. 分布式事务一致性
- **挑战**：跨服务的数据一致性问题难以解决
- **解决方案**：采用事件驱动架构，通过最终一致性保证数据同步

### 4. 容器安全配置
- **挑战**：容器化部署的安全性要求较高
- **解决方案**：通过证书管理和非root用户配置提升容器安全性
