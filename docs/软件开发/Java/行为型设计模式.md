# 行为型设计模式

> 来源：GOF设计模式 | 标签：设计模式 / Java

---

## 策略模式

**定义**：一系列算法封装起来，使它们可相互替换。

```java
// 运行环境
class Context {
    Strategy strategy;
    void setStrategy(Strategy s) { this.strategy = s; }
    void execute() { strategy.algorithm(); }
}

// 应用实例：诸葛亮的锦囊妙计、旅行出行方式
```

---

## 模板模式

**定义**：定义算法骨架，某些步骤延迟到子类实现。

```java
abstract class Game {
    abstract void init();
    abstract void start();
    abstract void end();
    public final void play() { init(); start(); end(); }
}
// 应用实例：spring-hibernate支持、房子建造流程
```

## 观察者模式

**定义**：一对多依赖关系，当一个对象状态改变，所有依赖对象自动通知。

```java
interface Observer { void update(); }
class Subject {
    List<Observer> observers = new ArrayList<>();
    void attach(Observer o) { observers.add(o); }
    void notifyAll() { observers.forEach(Observer::update); }
}
// 应用实例：拍卖师观察最高标价、股票价格变动通知
```

## 责任链模式

**定义**：请求沿着处理者链传递，直到被处理。

```java
interface Handler { void handleRequest(Request req); }
class HandlerA implements Handler {
    Handler next;
    void handleRequest(Request req) {
        if (canHandle(req)) doHandle(req);
        else if (next != null) next.handleRequest(req);
    }
}
// 应用实例：击鼓传花、过滤器链
```

## 命令模式

**定义**：请求封装为对象，可参数化、排队、撤销。

```java
interface Command { void execute(); }
class LightOnCommand implements Command {
    Light light;
    void execute() { light.on(); }
}
// 应用实例：遥控器按钮、任务队列
```

## 迭代器模式

**定义**：顺序访问集合元素，无须暴露底层表示。

```java
interface Iterator { boolean hasNext(); Object next(); }
class ArrayListIterator implements Iterator { /* ... */ }
// Java中的iterator就是迭代器模式
```

---

## 备忘录模式

**定义**：保存对象状态，可恢复到之前状态。

```java
class Originator { Memento save(); void restore(Memento m); }
class Memento { private State state; }
// 应用实例：游戏存档、Windows撤销
```

---

## 状态模式

**定义**：内部状态改变时改变行为，对象看起来修改了类。

```java
class Context { State state; void request() { state.handle(); } }
interface State { void handle(); }
// 应用实例：运动员状态（正常/超常）、订单状态
```

---

## 中介者模式

**定义**：用一个中介对象封装对象交互，降低耦合。

```java
interface Mediator { void notify(Colleague c); }
class ConcreteMediator implements Mediator {
    ColleagueA a; ColleagueB b;
    void notify(Colleague c) { /* 协调处理 */ }
}
// 应用实例：WTO贸易、机场调度、MVC的Controller
```

---

## 访问者模式

**定义**：将数据结构与操作分离，操作可独立变化。

```java
interface Visitor { void visit(ElementA a); void visit(ElementB b); }
interface Element { void accept(Visitor v); }
// 应用实例：编译器、报表生成
```

---

## 解释器模式

**定义**：定义语法解释器，解释特定语言。

```java
interface Expression { boolean interpret(Context context); }
// 应用实例：SQL解析、数学表达式计算
```
