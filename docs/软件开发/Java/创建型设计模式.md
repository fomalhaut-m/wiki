# 创建型设计模式

> 来源：GOF设计模式 | 标签：设计模式 / Java

---

## 工厂模式

**定义**：创建对象时暴露创建逻辑，通过共同接口指向新对象。

```java
interface Person { void println(); }
class Athlete implements Person { public void println() { /* ... */ } }

class PersonFactory {
    Person getPerson(String type) {
        if ("athlete".equals(type)) return new Athlete();
        // ...
    }
}
```

**应用**：提车 — 不需要知道车怎么造。

---

## 抽象工厂模式

**定义**：超级工厂创建其他工厂，提供一系列相关对象。

```java
interface Factory { ProductA createA(); ProductB createB(); }
class HomeFactory implements Factory {
    ProductA createA() { return new HomeProductA(); }
    ProductB createB() { return new HomeProductB(); }
}
```

**应用**：家庭衣柜 — 商务男装/女装、时尚男装/女装 成套。

---

## 单例模式

**定义**：保证一个类只有一个实例，提供全局访问点。

**实现**：饿汉式、懒汉式、双检锁、枚举

**应用**：班级只有一个班主任。

---

## 建造者模式

**定义**：分步骤构建复杂对象。

```java
class User {
    private String name;
    private int age;
    private User(UserBuilder b) { this.name = b.name; this.age = b.age; }
    static class UserBuilder {
        User build() { return new User(this); }
    }
}
```

**应用**：肯德基套餐、Java StringBuilder。

---

## 原型模式

**定义**：克隆已有对象创建新对象，避免高代价创建。

```java
class Prototype implements Cloneable {
    public Object clone() { return super.clone(); }
}
```

**应用**：细胞分裂、Java Object.clone()
