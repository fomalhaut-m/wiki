# 并发编程：线程安全与锁

> 来源：Java并发编程笔记 | 标签：Java / 并发

---

## 可见性

多个线程访问共享的可变状态时，需要正确管理。

> 没有同步时，编译器、处理器可能对操作执行顺序进行意想不到的调整。

### 失效数据

```java
// 线程不安全
public class MutableInteger {
    private int value;
    public int get() { return value; }
    public void set(int value) { this.value = value; }
}
```

一个线程调用set，另一个线程调用get，**可能看不到最新值**。

## volatile 关键字

比`synchronized`更轻量的同步机制。

### 特点

- 写入volatile相当于退出同步代码块
- 读取volatile相当于进入同步代码块
- **能保证可见性，但不能保证原子性**

```java
volatile int count = 0;
count++;  // 不安全！volatile不能保证原子性
```

### 适用条件

1. 对变量的写入操作不依赖当前值
2. 不与其他状态变量一起纳入不变性条件
3. 访问时不需要加锁

## 64位操作

`double`和`long`是64位的，非volatile时JVM允许拆成两个32位操作。

> 多线程共享`long`/`double`又不安全，必须用`volatile`或加锁保护。

## 发布与逸出

| 概念 | 说明 |
|------|------|
| **发布** | 使对象能在当前作用域之外使用 |
| **逸出** | 不该发布的内容被发布 |

```java
// 逸出示例
class ThisEscape {
    ThisEscape(EventSource source) {
        source.registerListener(new EventListener() {
            public void onEvent(Event e) {
                // 逸出！ThisEscape还未构造完成
            }
        });
    }
}
```

## ThreadLocal

每个线程独立的变量副本：

```java
ThreadLocal<String> threadLocal = new ThreadLocal<>();
threadLocal.set("本地数据");
String value = threadLocal.get();  // 只有当前线程能获取
```

适用场景：数据库连接、Session管理等。
