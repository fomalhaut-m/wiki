# 设计模式概述

> 来源：GOF《设计模式》| 标签：设计模式 / Java

---

## 什么是设计模式

设计模式是软件开发人员经过长时间试验和错误总结出来的最佳实践，是一般问题的**解决方案**。

1994年由 Erich Gamma、Richard Helm、Ralph Johnson、John Vlissides 四人合著出版《Design Patterns》，被称为 **GOF（四人帮）**。

---

## 设计模式类型

### 创建型模式（5个）

| 模式 | 说明 |
|------|------|
| **工厂模式** | 创建对象而不暴露创建逻辑 |
| **抽象工厂** | 创建一系列相关对象而不指定具体类 |
| **单例模式** | 保证一个类只有一个实例 |
| **建造者模式** | 分步骤构建复杂对象 |
| **原型模式** | 通过克隆现有对象创建新对象 |

### 结构型模式（8个）

| 模式 | 说明 |
|------|------|
| **适配器** | 将一个类的接口转换成客户端期望的格式 |
| **桥接** | 分离抽象部分和实现部分 |
| **组合** | 将对象组合成树形结构 |
| **装饰器** | 动态给对象添加额外职责 |
| **外观** | 提供统一的高层接口 |
| **享元** | 共享大量细粒度对象 |
| **代理** | 为另一个对象提供替身 |

### 行为型模式（12个）

| 模式 | 说明 |
|------|------|
| **责任链** | 请求沿着处理者链传递 |
| **命令** | 将请求封装为对象 |
| **迭代器** | 顺序访问集合元素 |
| **中介者** | 对象间通信通过中介者 |
| **备忘录** | 保存对象状态 |
| **观察者** | 一对多依赖关系 |
| **状态** | 对象内部状态改变时改变行为 |
| **策略** | 定义一系列算法 |
| **模板方法** | 定义算法骨架 |
| **访问者** | 操作分离 |

## 7大设计原则

| 原则 | 核心思想 |
|------|----------|
| **单一职责** | 一个类只负责一项职责 |
| **开闭原则** | 对扩展开放，对修改关闭 |
| **里氏替换** | 子类不破坏父类功能 |
| **依赖倒置** | 面向接口编程 |
| **接口隔离** | 最小接口依赖 |
| **迪米特法则** | 降低类间耦合 |
| **组合复用** | 优先使用组合而非继承 |

---

## 核心思想

1. **找出可能变化之处**，把它独立出来
2. **针对接口编程**，而不是针对实现编程
3. **松耦合设计**，对象间交互要灵活
