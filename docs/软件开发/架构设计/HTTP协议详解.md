# HTTP 报文详解

> 来源：HTTP协议笔记 | 标签：网络 / HTTP

---

## 报文流向

| 术语 | 说明 |
|------|------|
| **流入** | 报文流向源端服务器 |
| **流出** | 报文从服务器返回客户端 |
| **上游/下游** | 发送者在接收者的上游 |

> 所有报文的发送者都在接收者的**下游**

---

## 报文组成

```
起始行(start line) + 首部(header) + 主体(body)
```

### 请求报文格式

```
<method> <request-URL> <version>
<headers>

<entity-body>
```

### 响应报文格式

```
<version> <status> <reason-phrase>
<headers>

<entity-body>
```

---

## 首部分类

| 类型 | 说明 |
|------|------|
| **通用首部** | 请求和响应都能用 |
| **请求首部** | 含有请求信息 |
| **响应首部** | 含有响应信息 |
| **实体首部** | 描述实体内容 |

---

## 常见请求头

| 首部 | 说明 |
|------|------|
| Host | 目标主机 |
| User-Agent | 客户端信息 |
| Accept | 可接受的媒体类型 |
| Accept-Language | 可接受的语言 |
| Accept-Encoding | 可接受的编码 |
| Authorization | 认证信息 |

---

## 常见响应头

| 首部 | 说明 |
|------|------|
| Server | 服务器信息 |
| Content-Type | 内容类型 |
| Content-Length | 内容长度 |
| Content-Encoding | 编码方式 |
| Set-Cookie | 设置Cookie |
| Cache-Control | 缓存控制 |

---

## 状态码

| 类别 | 含义 |
|------|------|
| 1xx | 信息性 |
| 2xx | 成功 |
| 3xx | 重定向 |
| 4xx | 客户端错误 |
| 5xx | 服务器错误 |

---

## HTTP/2 改进

- **多路复用**：多个请求共享一个TCP连接
- **服务器推送**：服务端主动推送资源
- **头部压缩**：HPACK算法压缩
- **二进制格式**：更高效解析
