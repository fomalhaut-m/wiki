# 限流方案详解

> 来源：限流笔记 | 标签：架构 / 高并发

---

## 三把利器

| 方案 | 作用 |
|------|------|
| **缓存** | 提升速度，增大容量 |
| **降级** | 暂时屏蔽问题接口 |
| **限流** | 限制并发/请求速率 |

---

## 限流算法

### 漏桶算法

- 水（请求）进入漏桶，以固定速度出水
- 流入过快则溢出，直接拒绝
- **缺点**：无法处理突发流量

### 令牌桶算法

- 恒定速度往桶里放入令牌
- 请求需要获取令牌才能处理
- 桶无令牌则拒绝服务
- **优点**：支持突发流量，可方便调整速率

> QPS = 每秒查询率
> TPS = 每秒事务数
> 一次页面访问 = 1 TPS，但可能产生多次服务器请求 = N Qps

## 分布式限流

**Redis + 拦截器 + 自定义注解**（单体和分布式均适用）

```java
// Guava LoadingCache 实现滑动窗口限流
LoadingCache<Long, AtomicLong> counter = CacheBuilder.newBuilder()
    .expireAfterWrite(2, TimeUnit.SECONDS)
    .build(...);

public ResponseEntity getData() {
    long currentSeconds = System.currentTimeMillis() / 1000;
    if (counter.get(currentSeconds).incrementAndGet() > 50) {
        return ResponseEntity.status(429).body("访问速率过快");
    }
    // 业务处理
}
```

## Nginx 接入层限流

基于 `limit_req` 模块，采用漏桶算法。

| 限制方式 | 说明 |
|----------|------|
| **限制IP** | 根据客户端IP限速（比UserAgent更可靠） |
| **限制频率** | 基于时间窗口的请求数 |

```nginx
limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
location / {
    limit_req zone=one burst=5;
}
```

## 限流策略

| 策略 | 行为 |
|------|------|
| **拒绝服务** | 直接返回429/503 |
| **排队等待** | 请求进入队列等待 |
| **降级** | 返回默认/兜底数据 |
