## 三把利器

| 利器 | 目的 |
|------|------|
| **缓存** | 提升访问速度，增大处理容量 |
| **降级** | 服务问题时暂时屏蔽，待恢复 |
| **限流** | 限制并发访问/请求速率 |

---

## 限流算法

### 漏桶算法

```
水(请求) → 漏桶(以固定速度出水) → 溢出拒绝
```

**特点**：强行限制数据传输速率，适合匀速场景。

### 令牌桶算法

```
令牌桶(恒定速度放入令牌) → 请求获取令牌 → 无令牌则拒绝
```

**特点**：
- 允许一定程度的突发传输
- 可方便改变速率
- QPS/TPS 控制

---

## QPS vs TPS

| 指标 | 说明 |
|------|------|
| **QPS** | Queries Per Second，每秒查询数 |
| **TPS** | Transactions Per Second，每秒事务数 |

> 访问一个页面请求3次服务器 → 1个TPS，3个QPS

## 应用级限流

### 1. 控制并发数量
使用信号量机制（如Java的Semaphore）。

### 2. 控制访问速率
Guava RateLimiter + 令牌桶算法。

### 3. 控制时间窗口请求数
```java
LoadingCache<Long, AtomicLong> counter = 
    CacheBuilder.newBuilder()
        .expireAfterWrite(2, TimeUnit.SECONDS)
        .build();

public ResponseEntity getData() {
    long currentSeconds = System.currentTimeMillis() / 1000;
    if (counter.get(currentSeconds).incrementAndGet() > 50) {
        return ResponseEntity.status(429).build(); // 限流
    }
    // 业务处理
}
```

## 接入层限流（Nginx）

使用 `limit_req` 模块，基于IP限制访问频率。

```nginx
limit_req_zone $binary_remote_addr zone=ip_limit:10m rate=10r/s;
location / {
    limit_req zone=ip_limit burst=20;
}
```

**特点**：
- 用IP比UserAgent更可靠（IP无法伪造）
- 采用漏桶算法
