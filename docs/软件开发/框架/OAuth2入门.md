# OAuth 2.0 入门

> 来源：OAuth笔记 | 标签：安全 / 认证

---

## 概念

OAuth 2.0（RFC 6749）引入**授权层**，分离两种角色：
- **资源所有者**（用户）
- **客户端**（第三方应用）

用户同意后，资源服务器向客户端颁发**令牌**，客户端通过令牌请求数据。

---

## 四种授权方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **授权码** | 最安全，适合后端应用 | Web服务器 |
| **隐藏式** | 简化版，有安全风险 | 纯前端SPA |
| **密码式** | 直接交出用户名密码 | 高度信任应用 |
| **客户端凭证** | 以应用身份而非用户身份 | 服务间调用 |

---

## 授权码模式（最安全）

```
用户 → 应用 → 授权服务器 → 用户确认 → 返回授权码 → 应用 → 换取令牌
```

步骤：
1. 用户访问应用
2. 应用重定向到授权服务器
3. 用户确认授权
4. 授权服务器返回授权码
5. 应用用授权码换取访问令牌

---

## 前置条件

第三方应用必须先**备案**，获取两个身份识别码：
- **client_id** — 客户端ID
- **client_secret** — 客户端密钥

---

## 令牌类型

| 类型 | 说明 |
|------|------|
| **access_token** | 访问令牌，请求资源 |
| **refresh_token** | 刷新令牌，获取新access_token |
| **expires_in** | 过期时间（秒） |

---

## 常见场景

- 第三方登录（微信、Google登录）
- 授权访问用户数据（读取通讯录、日历）
- 微服务间调用
