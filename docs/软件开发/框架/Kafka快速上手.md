## 什么是Kafka

Kafka是分布式、基于**发布/订阅模式**的**消息队列**，主要用于大数据实时处理领域。

---

## 消息队列两种模式

### 点对点模式

- 生产者 → Queue → 消费者
- 消息被消费后，Queue中不再存储
- 一个消息只有一个消费者

### 发布/订阅模式

- 生产者 → Topic → 多个消费者
- 消息被所有订阅者消费

---

## Kafka 基础架构

| 组件 | 说明 |
|------|------|
| **Producer** | 消息生产者，向broker发送消息 |
| **Consumer** | 消息消费者，从broker取消息 |
| **Consumer Group** | 消费者组，组内每个消费者负责不同分区 |
| **Broker** | Kafka服务器，一台一个broker |
| **Topic** | 数据写入的基本单元，一个队列 |
| **Partition** | 分区，实现扩展性，一个Topic可分多区 |
| **Leader** | 主副本，负责读写 |
| **Follower** | 从副本，实时同步 |
| **Zookeeper** | 注册中心（0.9前offset存这，之后存本地） |

> Kafka消息默认存储7天（168小时）

## 使用好处

| 好处 | 说明 |
|------|------|
| **解耦** | 独立扩展或修改两边处理过程 |
| **可恢复性** | 部分组件失效不影响整个系统 |
| **缓冲** | 控制数据流经过系统的速度 |
| **峰值处理** | 顶住突发访问压力 |
| **异步通信** | 不需要立即处理消息 |

## 核心概念

### 分区与副本

```
Topic → Partition 0 (Leader) → Partition 1 (Leader) → Partition 2 (Leader)
              ↓                        ↓                      ↓
         Follower                 Follower               Follower
```

- 一个Partition可以有多个副本
- Leader故障时，某个Follower成为新的Leader

### 消费者组

- 消费者组内每个消费者负责不同分区的数据
- 一个分区只能有一个组内消费者消费
- 消费者组之间互不影响
