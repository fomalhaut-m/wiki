## 什么是Zookeeper

Zookeeper是分布式协调服务，用于管理大型主机。在分布式环境中协调和管理服务是一个复杂的过程，Zookeeper通过简单的架构和API解决了这个问题。

---

## 分布式应用挑战

| 挑战 | 说明 |
|------|------|
| **竞争条件** | 多机器竞争执行只需单台完成的任务 |
| **死锁** | 多操作相互等待无限期完成 |
| **不一致** | 数据部分失败 |

Zookeeper用**故障安全同步**和**原子性**来处理这些问题。

---

## Zookeeper 特点

1. **一个Leader + 多个Follower** 组成的集群
2. **半数以上节点存活**，集群就能正常服务
3. **全局数据一致**，每个Server保存相同数据副本
4. **更新请求顺序执行**
5. **原子性**，更新要么成功要么失败
6. **实时性**，Client能读到最新数据

---

## 数据结构

Zookeeper数据结构与Unix文件系统类似：

```
树结构
└── /root
    ├── /node1 (ZNode)
    ├── /node2
    └── /node3
```

- 每个ZNode默认存储**1MB**数据
- 每个ZNode通过**路径唯一标识**

---

## 应用场景

| 场景 | 说明 |
|------|------|
| **命名服务** | 按名称标识集群节点（类似DNS） |
| **配置管理** | 集中管理配置信息 |
| **集群管理** | 节点动态加入/离开 |
| **选举算法** | 选举Leader |
| **锁和同步** | 分布式锁 |
| **高可靠注册表** | 数据冗余存储 |

---

## 常用命令

```bash
# 启动Zookeeper
zkServer.sh start

# 客户端连接
zkCli.sh -server localhost:2181

# 查看节点
ls /

# 创建节点
create /mynode "data"

# 获取数据
get /mynode

# 删除节点
delete /mynode
```
