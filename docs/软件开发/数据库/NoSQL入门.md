# NoSQL 入门

> 来源：NoSQL笔记 | 标签：数据库 / NoSQL

---

## 什么是NoSQL

NoSQL = Not Only SQL，不仅仅是SQL。泛指非关系型数据库。

**解决的问题**：
- 大规模数据集合多重数据种类
- 超大规模数据存储
- web2.0网站的高并发需求

---

## 发展历程

| 阶段 | 技术 | 问题 |
|------|------|------|
| 1. 单机MySQL | 直接读写 | 数据量、索引、访问量瓶颈 |
| 2. Memcached缓存 | 文件缓存 → Memcached | 多机无法共享缓存 |
| 3. 主从读写分离 | MySQL master-slave | 读写集中一个库 |
| 4. 分表分库集群 | MySQL Cluster | 表锁、高并发下性能下降 |
| 5. NoSQL | 非关系型数据库 | 扩展性、IO压力 |

---

## NoSQL优势

| 优势 | 说明 |
|------|------|
| **易扩展** | 数据无关系，架构可扩展 |
| **大数据量高性能** | 无关系型结构，细粒度Cache |
| **灵活数据类型** | 无需预定义字段，随时存储 |
| **高可用** | 副本集、集群保证 |

---

## RDBMS vs NoSQL

| 对比 | RDBMS | NoSQL |
|------|-------|-------|
| 数据结构 | 结构化 | 非结构化 |
| 查询语言 | SQL | 无声明性查询 |
| 模式 | 预定义 | 无预定义模式 |
| 存储方式 | 表+关系 | KV、列、文档、图 |
| 一致性 | ACID | 最终一致性 |
| 扩展方式 | 垂直扩展 | 水平扩展 |

---

## NoSQL分类

| 类型 | 代表 | 适用场景 |
|------|------|----------|
| **KV存储** | Redis, Memcached | 缓存、 session |
| **列存储** | HBase, Cassandra | 大数据存储 |
| **文档存储** | MongoDB | JSON文档 |
| **图形存储** | Neo4j | 社交网络、推荐 |
