# 反射与代理

## 反射（Reflection）

运行期动态获取类的信息并操作。

### 获取 Class 对象

```java
// 方式一：类名.class
Class<?> c1 = User.class;

// 方式二：对象.getClass()
Class<?> c2 = user.getClass();

// 方式三：Class.forName()
Class<?> c3 = Class.forName("com.example.User");
```

### 常用方法

```java
Class<?> c = Class.forName("com.example.User");

c.getName()                    // 全限定类名
c.getSimpleName()             // 类名
c.getPackage()                 // 包信息
c.getSuperclass()             // 父类
c.getInterfaces()             // 实现的接口
c.getFields()                 // 公开字段（含继承）
c.getDeclaredFields()          // 本类所有字段
c.getMethods()                 // 公开方法
c.getDeclaredMethods()         // 本类所有方法
c.getConstructors()           // 公开构造器
c.getDeclaredConstructors()    // 所有构造器

// 创建实例
Object obj = c.newInstance();  // 调用无参构造器

// 获取构造器
Constructor<?> con = c.getConstructor(String.class, int.class);
Object obj = con.newInstance("luke", 26);
```

### 操作字段

```java
Field nameField = c.getDeclaredField("name");
nameField.setAccessible(true);  // 设为可访问（private 也行）
nameField.set(obj, "luke");    // 写入
String name = (String) nameField.get(obj);  // 读取
```

### 操作方法

```java
Method setName = c.getDeclaredMethod("setName", String.class);
setName.setAccessible(true);
setName.invoke(obj, "luke");    // 调用

Method getName = c.getDeclaredMethod("getName");
getName.setAccessible(true);
String result = (String) getName.invoke(obj);
```

## ClassLoader（类加载器）

三层类加载器：

| 加载器 | 负责路径 |
|--------|---------|
| Bootstrap ClassLoader | `JAVA_HOME/jre/lib` |
| Ext ClassLoader | `JAVA_HOME/jre/lib/ext` |
| App ClassLoader | classpath（`java.class.path`） |

### 双亲委派模型

类加载请求逐层向上传递，直到 Bootstrap ClassLoader，只有父加载器找不到时才由自己加载。

```
App → Ext → Bootstrap → 找到？Bootstrap加载 | 找不到？Ext加载 | 找不到？App加载
```

**目的：** 避免类的重复加载 + 保证核心类安全（不被自定义类替换）。

## 代理（Proxy）

不修改原类的情况下，增强方法。

### 静态代理

手动编写代理类，实现相同接口：

```java
public class UserServiceProxy implements UserService {
    private UserService target;  // 被代理对象

    public UserServiceProxy(UserService target) {
        this.target = target;
    }

    @Override
    public void save() {
        System.out.println("=== 前置增强 ===");
        target.save();
        System.out.println("=== 后置增强 ===");
    }
}
```

### 动态代理（Proxy.newProxyInstance）

运行时生成代理类，无需手动写：

```java
UserService target = new UserServiceImpl();

UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(),    // 类加载器
    target.getClass().getInterfaces(),     // 被代理类实现的接口
    (proxy, method, args) -> {            // InvocationHandler
        System.out.println("=== 前置增强 ===");
        Object result = method.invoke(target, args);
        System.out.println("=== 后置增强 ===");
        return result;
    }
);

proxy.save();
```

### CGLIB 动态代理

代理类不实现接口，通过继承被代理类生成子类：

```java
public class UserService {
    public void save() { System.out.println("save"); }
}

// CGLIB 代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(UserService.class);
enhancer.setCallback((MethodInterceptor) (obj, method, args, proxy) -> {
    System.out.println("=== 前置 ===");
    Object result = proxy.invokeSuper(obj, args);
    System.out.println("=== 后置 ===");
    return result;
});
UserService proxy = (UserService) enhancer.create();
proxy.save();
```

| 对比 | JDK 动态代理 | CGLIB |
|------|------------|-------|
| 要求 | 被代理类必须实现接口 | 不需要（继承方式） |
| 原理 | Proxy 类 + InvocationHandler | 继承被代理类 |
| 性能 | 略慢（反射调用） | 更快（生成字节码） |
| 场景 | 通用代理 | 代理无接口类 |

## 注解（Annotation）

注解是元数据，不影响程序逻辑，但可被编译器/框架读取。

### 内置注解

```java
@Override       // 编译期检查
@Deprecated    // 标记废弃
@SuppressWarnings("unchecked")  // 抑制警告
@FunctionalInterface  // 函数式接口检查
```

### 自定义注解

```java
@Retention(RetentionPolicy.RUNTIME)  // 运行时保留
@Target(ElementType.METHOD)          // 作用在方法上
public @interface MyAnnotation {
    String value() default "默认值";  // 参数
    int level() default 0;
}
```

### 反射读取注解

```java
Method m = c.getDeclaredMethod("methodName");
MyAnnotation annotation = m.getAnnotation(MyAnnotation.class);
if (annotation != null) {
    String val = annotation.value();
    int lvl = annotation.level();
}
```

### 注解的用途

- **框架配置**：Spring `@Component`、`@Autowired`
- **文档生成**：Javadoc 工具
- **编译检查**：`@Override`、`@Deprecated`
- **AOP**：切面编程的核心
