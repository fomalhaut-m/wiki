# 并发编程

## 线程基础

### 创建线程

```java
// 方式一：继承 Thread
class MyThread extends Thread {
    @Override
    public void run() { /* ... */ }
}
new MyThread().start();

// 方式二：实现 Runnable（推荐）
class MyRunnable implements Runnable {
    @Override
    public void run() { /* ... */ }
}
new Thread(new MyRunnable()).start();

// 方式三：Callable + FutureTask
FutureTask<String> task = new FutureTask<>(() -> "result");
new Thread(task).start();
String result = task.get();  // 阻塞等待
```

### 线程状态

```
NEW → RUNNABLE → BLOCKED/WAITING → TERMINATED
```

| 状态 | 说明 |
|------|------|
| NEW | 刚创建，未 start |
| RUNNABLE | 运行中或就绪 |
| BLOCKED | 等待获取锁 |
| WAITING | wait / join / park |
| TIMED_WAITING | sleep / wait(n) / join(n) / parkNanos |
| TERMINATED | 执行完毕 |

### 常用方法

```java
thread.start()           // 启动线程
thread.join()            // 等待该线程执行完毕
thread.join(millis)      // 最多等 millis 毫秒
Thread.sleep(millis)     // 休眠（不释放锁）
Thread.yield()           // 礼让线程
thread.interrupt()       // 设置中断标志
thread.isInterrupted()    // 检查是否被中断
Thread.currentThread()   // 获取当前线程
```

### 守护线程

```java
thread.setDaemon(true);  // 设置为守护线程（finally 不一定执行）
// JVM 不等待守护线程退出就终止
```

## JUC 并发包

### ReentrantLock（可重入锁）

`synchronized` 的增强版，提供公平锁/非公平锁、可中断锁等待：

```java
private final ReentrantLock lock = new ReentrantLock();

// 标准模式
lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();  // 必须 finally 释放
}

// 可中断锁等待
lock.lockInterruptibly();

// 尝试获取（不等）
if (lock.tryLock(5, TimeUnit.SECONDS)) {
    try { /* ... */ }
    finally { lock.unlock(); }
}
```

**公平锁 vs 非公平锁：**
```java
ReentrantLock fairLock = new ReentrantLock(true);  // 公平
ReentrantLock unfairLock = new ReentrantLock();       // 非公平（默认）
```

### Condition（条件变量）

`synchronized` + `Object.wait/notify` 的替代：

```java
private final ReentrantLock lock = new ReentrantLock();
private final Condition condition = lock.newCondition();

lock.lock();
try {
    condition.await();           // 等候
    condition.signal();          // 唤醒一个
    condition.signalAll();      // 唤醒全部
} finally {
    lock.unlock();
}
```

### Semaphore（信号量）

控制同时访问资源的线程数：

```java
Semaphore semaphore = new Semaphore(5);  // 5 个许可
semaphore.acquire();                      // 获取一个许可
semaphore.release();                      // 归还许可
```

### CountDownLatch（倒计时门闩）

主线程等待 N 个子线程执行完毕：

```java
CountDownLatch latch = new CountDownLatch(10);
latch.countDown();   // 子线程调用，减 1
latch.await();       // 主线程等待
```

### CyclicBarrier（循环栅栏）

N 个线程互相等待，全部就绪后一起执行：

```java
CyclicBarrier barrier = new CyclicBarrier(10, () -> System.out.println("全部到达"));
barrier.await();  // 每个线程调用，等 10 个都到了才放行
```

### ReadWriteLock（读写锁）

读-读不互斥，读-写/写-写互斥：

```java
ReadWriteLock rwLock = new ReentrantReadWriteLock();
Lock readLock = rwLock.readLock();
Lock writeLock = rwLock.writeLock();
```

### LockSupport

更底层的线程阻塞/唤醒工具，不需要获取锁：

```java
LockSupport.park();        // 阻塞当前线程
LockSupport.unpark(thread); // 唤醒指定线程
```

### ConcurrentHashMap（JDK 1.8+）

```java
// 不使用 synchronized，整个表用 CAS + synchronized
ConcurrentHashMap<K, V> map = new ConcurrentHashMap<>();

map.putIfAbsent(key, value);  // 原子操作：不存在才插入
map.computeIfAbsent(key, k -> new Object());  // 原子计算
```

### BlockingQueue（阻塞队列）

| 方法 | 说明 |
|------|------|
| `put(e)` | 队列满则阻塞 |
| `take()` | 队列空则阻塞 |
| `offer(e, timeout)` | 超时则放弃 |
| `poll(timeout)` | 超时则返回 null |

## 线程池

### 常用线程池

```java
Executors.newFixedThreadPool(10);         // 固定大小
Executors.newCachedThreadPool();           // 弹性缓存
Executors.newSingleThreadExecutor();       // 单线程
Executors.newScheduledThreadPool(5);      // 定时调度
```

### 手写线程池参数

```java
new ThreadPoolExecutor(
    corePoolSize,       // 核心线程数
    maxPoolSize,        // 最大线程数
    keepAliveTime,      // 空闲线程存活时间
    TimeUnit,           // 时间单位
    workQueue,          // 任务队列
    threadFactory,      // 线程工厂
    handler             // 拒绝策略
);
```

### 拒绝策略

| 策略 | 行为 |
|------|------|
| `AbortPolicy` | 抛 RejectedExecutionException（默认） |
| `CallerRunsPolicy` | 由调用方线程执行 |
| `DiscardPolicy` | 丢弃任务，不抛异常 |
| `DiscardOldestPolicy` | 丢弃最老的任务 |

## 线程安全

### 线程安全的本质

**可见性、原子性、有序性** — 简称"并发三性"。

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 可见性 | CPU 缓存导致 | `volatile`、`synchronized` |
| 原子性 | 上下文切换 | `synchronized`、JUC 原子类、`Lock` |
| 有序性 | 编译器和 CPU 重排序 | `volatile`、`synchronized` |

### volatile

保证**可见性**和**有序性**，但不保证原子性。

```java
volatile boolean flag = true;  // 线程间可见
```

### synchronized

修饰实例方法、静态方法、代码块，保证**原子性+可见性+有序性**。

```java
synchronized void method() { }           // 实例方法（锁 this）
synchronized static void method() { }    // 类方法（锁 .class）
synchronized (this) { }                  // 代码块（锁 this）
```

### CAS（Compare And Swap）

乐观锁，CPU 原子指令，无锁实现：

```java
// AtomicInteger 示例
AtomicInteger ai = new AtomicInteger(5);
ai.compareAndSet(5, 10);  // 期望是 5，才改成 10，返回 true
```

**CAS 的 ABA 问题：** 解决办法是用 `AtomicStampedReference`（版本号）。

## 死锁

```java
// 死锁示例
Thread A: lock1.lock() → lock2.lock()
Thread B: lock2.lock() → lock1.lock()

// 定位
jstack <pid>  // 查找 blocked 线程和锁信息
```

**避免死锁：** 按固定顺序加锁；用 `tryLock` 加超时。
