# 引用类型

Java 四种引用，从强到弱：

## 强引用（Strong Reference）

```java
Object obj = new Object();  // 强引用
obj = null;                 // 断开引用，GC 才能回收
```

**最常见，GC 不会回收强引用对象。**

## 软引用（Soft Reference）

内存不足时才回收，可用于**缓存**：

```java
SoftReference<Object> soft = new SoftReference<>(new Object());
Object obj = soft.get();  // 可能返回 null
```

**使用场景：** 图片缓存，内存不足时自动清理。

## 弱引用（Weak Reference）

GC 扫描到即回收，生命周期到下次 GC 为止：

```java
WeakReference<Object> weak = new WeakReference<>(new Object());
Object obj = weak.get();  // GC 后可能为 null
```

**使用场景：** `WeakHashMap` 的 Entry、ThreadLocal 的 Entry（防止内存泄漏的辅助手段）。

## 虚引用（Phantom Reference）

最弱引用，**无法通过 get() 获取对象**，仅用于跟踪对象被回收：

```java
PhantomReference<Object> phantom = new PhantomReference<>(new Object(), queue);
// get() 永远返回 null
// 对象被回收时，引用会被放入 ReferenceQueue
```

**使用场景：** 跟踪对象销毁、清理native 资源。

## 对比

| 引用类型 | GC 时机 | 用途 |
|---------|---------|------|
| 强引用 | 永不回收 | 普通对象 |
| 软引用 | 内存不足时 | 缓存 |
| 弱引用 | 下次 GC 时 | 缓存、防止内存泄漏 |
| 虚引用 | 随时可回收 | 跟踪对象回收 |
