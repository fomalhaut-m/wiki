# 多线程面试

## Volatile

保证**可见性**和**有序性**，不保证原子性。

```java
volatile boolean flag = true;  // 线程间立即可见
```

**原理：** 写操作后立即刷新到主内存；读操作直接从主内存读取。

## CAS（Compare And Swap）

乐观锁，CPU 原子指令，无锁实现：

```java
// AtomicInteger 内部原理
public final boolean compareAndSet(int expect, int update) {
    return unsafe.compareAndSwapInt(this, valueOffset, expect, update);
}
```

**ABA 问题：** A → B → A，值变了但无法检测。解决方案：`AtomicStampedReference`（加版本号）。

## synchronized

修饰方法或代码块，保证原子性+可见性+有序性：

```java
// 实例方法（锁 this）
public synchronized void method() { }

// 静态方法（锁 .class）
public static synchronized void staticMethod() { }

// 代码块（手动指定锁对象）
synchronized (this) { }
synchronized (obj) { }
```

## ReentrantLock

可重入锁，比 `synchronized` 更灵活：

```java
private final ReentrantLock lock = new ReentrantLock();

// tryLock() — 尝试获取，获取不到立即返回
if (lock.tryLock()) {
    try { /* ... */ } finally { lock.unlock(); }
}

// tryLock(timeout) — 可超时
lock.tryLock(5, TimeUnit.SECONDS)
```

**公平锁 vs 非公平锁：**
```java
ReentrantLock fairLock = new ReentrantLock(true);  // 公平（默认非公平）
```

## 线程池参数

```java
new ThreadPoolExecutor(
    corePoolSize,      // 核心线程数
    maxPoolSize,       // 最大线程数
    keepAliveTime,     // 空闲线程存活时间
    TimeUnit,          // 时间单位
    BlockingQueue,     // 任务队列
    ThreadFactory,     // 线程工厂
    RejectedExecutionHandler  // 拒绝策略
);
```

**拒绝策略：**

| 策略 | 行为 |
|------|------|
| AbortPolicy | 抛异常（默认） |
| CallerRunsPolicy | 由调用方线程执行 |
| DiscardPolicy | 丢弃新任务 |
| DiscardOldestPolicy | 丢弃最老的 |

**线程数设置公式：**
```
线程数 = CPU 核心数 × (1 + IO 耗时 / CPU 计算耗时)
```

## 阻塞队列

| 队列 | 特点 |
|------|------|
| `ArrayBlockingQueue` | 有界，数组实现 |
| `LinkedBlockingQueue` | 无界（Integer.MAX_VALUE） |
| `SynchronousQueue` | 容量为 0，必须同步交付 |
| `PriorityBlockingQueue` | 优先级队列 |

## 生产者-消费者问题

```java
// wait / notify 实现
public class Storage {
    private final int MAX = 10;
    private final List<Object> list = new ArrayList<>();

    public synchronized void produce(int n) {
        while (list.size() >= MAX) {
            this.wait();
        }
        list.add(n);
        System.out.println("生产: " + n);
        this.notifyAll();
    }

    public synchronized Object consume() {
        while (list.isEmpty()) {
            this.wait();
        }
        Object o = list.remove(0);
        System.out.println("消费: " + o);
        this.notifyAll();
        return o;
    }
}
```

## ThreadLocal

线程本地变量，每个线程独立副本：

```java
ThreadLocal<String> tl = new ThreadLocal<>();
tl.set("luke");          // 本线程设置
String val = tl.get();   // 本线程获取
tl.remove();             // 防止内存泄漏（建议 finally 中调用）
```

**内存泄漏风险：** ThreadLocalMap 的 Entry 继承 WeakReference，若 ThreadLocal 无强引用，GC 时 Entry 被回收，但 value 不回收（除非线程结束）。
