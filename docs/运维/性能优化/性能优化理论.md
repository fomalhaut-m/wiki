# 性能优化理论

## 系统性能的视角

### 负载视角
从客户端/用户的角度出发，关注响应时间、吞吐量。

### 资源视角
从系统内部出发，关注 CPU、内存、磁盘 I/O、网络等资源的利用情况。

## 性能分析挑战

### 1. 主观性
性能分析**没有绝对标准**，同一个指标好不好取决于具体场景。

**方法：** 将主观问题客观化 → 量化指标 + 具体场景。

### 2. 复杂性
性能问题往往出现在**子系统之间的互联**上，而非单个组件。

**方法：** 梳理清楚子系统间的关系，理解协同工作的方式。

### 3. 多因性
一个问题可能由**多个因素**叠加导致。

**方法：** 量化每个因素的贡献度，逐一排查。

### 4. 多问题性
复杂系统中通常**同时存在多个性能问题**。

**方法：** 量化问题程度，优先解决影响最大的。

## 核心指标

### 延时（Latency）
系统处理请求的**等待时间**。是最重要的性能指标之一。

```
响应时间 = 网络延迟 + 服务处理时间 + 传输延迟
```

### 吞吐量（Throughput）
单位时间内处理的请求数。

```
QPS = Queries Per Second
TPS = Transactions Per Second
```

### 资源利用率
CPU、内存、磁盘、网络的使用程度。

## 可观测性工具

| 类型 | 工具 | 说明 |
|------|------|------|
| **计数器** | `iostat`, `vmstat`, `netstat` | 资源使用统计 |
| **剖析** | `top`, `htop`, `perf` | 采样分析 CPU 热点 |
| **跟踪** | `strace`, `BPF`, `dtrace` | 追踪系统/应用调用 |

### Linux 常用观测命令

```bash
top              # CPU 和进程实时状态
htop             # top 的增强版（更直观）
vmstat 1         # 每秒刷新虚拟内存统计
iostat -x 1      # 磁盘 I/O 统计
netstat -tulnp   # 网络连接和端口
sar -u 1         # CPU 使用率历史
sar -r 1         # 内存使用率历史
```

## 性能优化的基本步骤

1. **设定目标**：明确的性能指标（如：接口 < 200ms）
2. **建立基准**：测量当前性能水平
3. **定位瓶颈**：用工具找到最拖后腿的子系统
4. **优化**：针对瓶颈下手
5. **验证**：优化后重新测量，确认提升

## 性能优化的常见方向

| 方向 | 说明 |
|------|------|
| **CPU** | 代码算法优化、减少上下文切换 |
| **内存** | 减少内存分配、JVM 调优 |
| **I/O** | 磁盘顺序读写、使用缓存 |
| **网络** | 连接池、压缩、CDN |
| **数据库** | 索引优化、SQL 优化、读写分离 |
| **缓存** | 多级缓存、缓存穿透/击穿/雪崩 |
