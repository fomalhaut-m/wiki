# PMP 整合管理

> 来源：《项目管理知识体系》读书笔记 | 标签：PMP / 项目管理

---

## 概述

项目整合管理包括识别、定义、组合、统一和协调各过程，贯穿项目始终。

> 项目经理是**整合者**，在互相矛盾、互相竞争的目标中寻找最佳平衡点。

---

## 七大过程

| 过程 | 所属组 | 说明 |
|------|--------|------|
| 4.1 制定项目章程 | 启动 | 正式批准项目，授权项目经理使用组织资源 |
| 4.2 制定项目管理计划 | 规划 | 定义、准备、协调所有计划组成部分 |
| 4.3 指导与管理项目工作 | 执行 | 领导和执行项目管理计划中确定的工作 |
| 4.4 管理项目知识 | 执行 | 使用现有知识生成新知识，帮助组织学习 |
| 4.5 监控项目工作 | 监控 | 跟踪、审查、报告整体项目进展 |
| 4.6 实施整体变更控制 | 监控 | 审查、批准、管理所有变更 |
| 4.7 结束项目或阶段 | 收尾 | 终结所有活动 |

---

## 七句话记忆

1. **一定章程做授权**
2. **二定计划做方案**
3. **三定执行做指导**
4. **四定知识做更新**
5. **五定监控做跟踪**
6. **六定变更做审批**
7. **七定收尾做结束**

---

## 整合管理特点

- **全生命周期** — 贯穿始终
- **综合性** — 整合各领域要素
- **全局性** — 协调统一

> 只要存在结合部，就需要整合。项目管理思维就是整合。
