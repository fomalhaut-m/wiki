# 工程实践规范

> 来源：团队项目规范笔记（cursor.rule.md） | 标签：工程规范 / Java

---

## DDD项目结构

```
com.anzir.xx.service.provider
├── domain           # 领域层（按业务划分）
├── api              # 接口层
├── config           # 配置层
└── exception        # 异常层
```

**命名规范：**
- 常量类：`XxxConsts`
- 实体类：`XxxEntity`
- 值对象：`XxxVo`
- 服务类：`XxxService`
- 配置类：`XxxConfig`
- 接口类：`XxxApi`

---

## API设计规范

- 路径格式：`/xx/{领域}/{操作}`
- 返回类型：`ServiceResponse<T>`
- 查询用 `@PostMapping` + `BizQueriesStatement`
- 新增/修改用 `@PostMapping` + `XxxRequest`
- 必须参数校验（`@Validated` + `@NotNull`）

---

## 异常处理

- 业务异常继承 `CustomException`
- 全局异常处理：`@RestControllerAdvice`
- 异常日志记录完整堆栈
- 区分检查型异常和运行时异常

---

## 日志规范

| 级别 | 用途 |
|------|------|
| `ERROR` | 错误日志，记录异常堆栈 |
| `WARN` | 警告日志，潜在问题 |
| `INFO` | 关键业务节点 |
| `DEBUG` | 调试信息 |

- 使用 `@Slf4j(topic = "xxx")` 指定日志主题
- 使用 `@LogElapsedTime` 监控性能
- 敏感信息（密码、手机号）不得明文记录

---

## Spring Boot规范

- 使用 `application.yml` 配置
- 使用 `@ConfigurationProperties` 实现类型安全配置
- 缓存统一在 `CacheConfig` 定义
- 使用Spring Profiles区分环境
- 敏感配置使用加密或环境变量

---

## 安全规范

- 密码使用BCrypt编码
- API接口权限控制（`@PreAuthorize`）
- 避免SQL注入（参数化查询）
- 避免XSS攻击（输入输出转义）
- 避免CSRF攻击（CSRF令牌）

---

## 测试规范

- 单元测试：JUnit 5 + Spring Boot Test
- Web层测试：MockMvc
- 测试命名：`test{方法名}_{场景}`
- 核心类必须有单元测试覆盖

---

## Lombok使用

| 注解 | 用途 |
|------|------|
| `@Data` | DTO/VO数据对象 |
| `@Value` | 不可变对象 |
| `@Builder` / `@SuperBuilder` | 复杂对象创建 |
| `@Getter` / `@Setter` | 实体类 |
| `@Slf4j` | 日志 |
| `@AllArgsConstructor` / `@NoArgsConstructor` | 构造函数 |

⚠️ 谨慎使用 `@EqualsAndHashCode`，避免循环引用
