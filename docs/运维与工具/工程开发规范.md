# 工程开发规范

> 来源：团队开发规范（cursor.rule.md）| 标签：工程规范

---

## 项目架构

- 遵循**领域驱动设计(DDD)**的项目结构规范
- 顶层包：`com.anzir.xx.service.provider`
- 二级包：`domain`、`api`、`config`、`exception`
- **domain包**按业务领域细分，如 `commodity`、`promotion`、`policy`
- 避免循环依赖，上层模块不依赖下层模块

---

## 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 包名 | 全小写，多个单词用`.`分隔 | `com.anzir.xx.service.domain.commodity` |
| 常量类 | `xxxConsts` | `SystemIdentityConsts` |
| 实体类 | `xxxEntity` | `CommodityEntity` |
| 值对象 | `xxxVo` | `AvailablePromotionsVo` |
| 服务类 | `xxxService` | `PromotionService` |
| 配置类 | `xxxConfig` | `CacheConfig` |
| 接口类 | `xxxApi` | `CommodityApi` |
| 类名 | 帕斯卡（PascalCase） | `UserController` |
| 方法/变量 | 驼峰（camelCase） | `findUserById` |
| 常量 | 全大写 | `MAX_RETRY_ATTEMPTS` |

---

## API 接口规范

- API类使用 `@RestController`，命名 `xxxApi`
- 路径格式：`/xx/{领域}/{操作}`，如 `/xx/commodity/query`
- 返回类型统一：`ServiceResponse<T>`
- 查询：`@PostMapping` + `BizQueriesStatement`
- 新增/修改：`@PostMapping` + `xxxRequest`
- 参数校验：`@Validated` + `@NotNull/@NotBlank`

---

## 异常处理

- 业务异常继承 `CustomException`
- 全局处理：`@RestControllerAdvice` + `@ExceptionHandler`
- 异常结果统一转换为 `ServiceResponse` 格式
- 业务方法不吞噬异常，必须记录日志或向上抛出

## 日志规范

| 级别 | 使用场景 |
|------|----------|
| `ERROR` | 错误日志，记录异常堆栈 |
| `WARN` | 警告日志，潜在问题 |
| `INFO` | 关键业务节点 |
| `DEBUG` | 调试信息 |

```java
@Slf4j(topic = "xxx")
public class XxxApi {
    @LogElapsedTime
    public ServiceResponse<?> query() {
        log.info("操作:{}，参数:{}", "query", param);
    }
}
```

## Lombok 使用

| 注解 | 适用场景 |
|------|----------|
| `@Data` | DTO/VO |
| `@Getter/@Setter` | 实体类 |
| `@Value` | 不可变对象 |
| `@Builder` | 复杂对象创建 |
| `@Slf4j` | 日志 |
| `@AllArgsConstructor` | 全参构造 |
| `@NoArgsConstructor` | 无参构造 |

> ⚠️ 避免在实体类同时使用 `@Data` 和 `@EqualsAndHashCode(callSuper=false)`

## 业务代码规范

- 单一职责原则，一个类只负责一个功能
- 服务层方法应具有原子性
- 避免循环中数据库操作，使用批处理
- 敏感操作添加分布式锁
- 查询结果使用缓存优化（`@Cacheable`）
- 优先使用构造器注入
- 使用 `@Async` 实现异步处理

---

## 安全规范

- 使用Spring Security认证授权
- 密码加密存储（BCrypt）
- `@PreAuthorize` 权限控制
- 敏感数据加密存储
- **防SQL注入**：参数化查询
- **防XSS**：输入输出转义
- **防CSRF**：CSRF令牌

## 测试规范

```java
// 测试类命名
class UserServiceTest {}

// 测试方法命名
void testFindUserById_存在用户() {}
```

- 单元测试独立、快速、可重复
- 使用 `@SpringBootTest` 集成测试
- 使用 `@MockMvc` 测试Web层
- 覆盖正常路径和异常路径
