## 冒烟测试

**冒烟测试**来源于硬件测试，是在软件开发过程中的初步测试方法，主要目的是快速验证软件基本功能是否正常。

### 目的

1. **快速发现严重问题** — 避免在后续测试中浪费时间和资源
2. **确定版本稳定性** — 基本功能稳定后才进行深入测试

### 实施过程

1. **确定测试用例** — 选择关键核心功能（登录、数据保存、主要业务流程）
2. **执行测试** — 手动或自动化快速执行
3. **结果判断** — 全部通过 = 通过，有失败 = 不通过

### 重要性

| 价值 | 说明 |
|------|------|
| 提高测试效率 | 早期筛选不稳定版本 |
| 降低测试成本 | 早期发现问题，修复成本低 |
| 保证软件质量 | 为后续全面测试提供保障 |

## 测试类型汇总

| 类型 | 说明 |
|------|------|
| **单元测试** | 对单个函数/方法测试 |
| **集成测试** | 模块间集成测试 |
| **系统测试** | 完整系统测试 |
| **冒烟测试** | 初步快速验证 |
| **回归测试** | 修改后确保没有引入新问题 |
| **压力测试** | 高负载下的性能 |
| **UAT** | 用户验收测试 |
