# 分词与词表

## 什么是分词（Tokenization）

将文本切割为更小的独立单元（Token），让计算机能解析文本结构与语义。

```
英文示例：
"I want to study ACA."
  ↓
['I', 'want', 'to', 'study', 'ACA', '.']

每个 token → 词表映射 → token id
最终以 token id 列表供模型处理
```

## 三种粒度的分词

| 粒度 | 方式 | 适用场景 | 优缺点 |
|------|------|---------|--------|
| **词粒度** | 以完整单词为单位 | 英文等西方语言 | 词表大，难处理新词 |
| **字符粒度** | 以单个汉字/字符为单位 | 中文等表意文字 | 序列长度长，语义弱 |
| **子词粒度** | 拆为词根、词缀等单元 | 新词、专有名词、网络用语 | 平衡词表大小和语义 |

## 常见分词算法

### BPE（Byte Pair Encoding）

GPT 系列使用。通过合并最高频的字符对构建词表。

```python
# 原始: ["look", "in", "look"]
# 合并: ["lo", "ok", "in", "lo", "ok"]
# 词表逐步扩大，平衡粒度
```

### WordPiece

BERT 使用。类似 BPE，但基于子词概率合并。

### SentencePiece

Google 开发，支持多语言，无需空格分词。

## 词表大小的影响

| 词表规模 | 特点 |
|---------|------|
| 小词表（~10k） | 序列短，表达丰富，但易欠拟合 |
| 大词表（~50k+） | 覆盖更多词，序列短，但参数量大 |

**GPT-4 的词表：** 100,276 tokens（用 BPE）

## 为什么分词很重要

1. **影响模型能记住的上下文长度**（固定 max_tokens 下，token 数越少消耗越慢）
2. **决定信息密度**（中文比英文 token 效率低）
3. **影响推理成本**（API 按 token 计费）

```
中文："你好世界" → 4 个汉字 → 约 2-4 个 token
英文："hello world" → 2 个词 → 约 2-4 个 token
```
