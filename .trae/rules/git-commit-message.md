# Git提交日志生成规则（与push-with-log.sh脚本默认规则完全匹配）
你可以直接修改以下规则，AI会优先遵守这里的配置，无需修改脚本。

## 默认规则（和脚本内置规则一致，无需修改）：
1. 输出必须是中文，语言自然流畅
2. 长度控制在100字以内，可以适当详细描述变更内容
3. 只输出日志内容，不要添加额外说明
4. 使用动宾结构，如：修复xxx问题、新增xxx功能、优化xxx性能
5. 严格基于输入内容生成，不得添加任何想象或虚构信息
6. 可以包含变更的范围和具体改动点，让日志更清晰

## 可选自定义示例（取消注释即可生效）：
# 6. 必须添加emoji前缀：✅新增/✨优化/🐛修复/⚡️重构
# 7. 必须添加范围标签：[文档]/[脚本]/[代码]/[配置]
# 8. 禁止使用英文缩写，全部用中文描述
# 9. 长度控制在100字以内, 较为详细描述变更内容
