# Luke's Wiki MkDocs 项目 Trae 开发规则
## 一、项目目录结构强制规范（严格遵循现有结构，禁止修改）
```
project-root/
├── .github/workflows/       # GitHub Actions 自动化部署配置
├── .idea/                    # IDE 配置文件，禁止提交到Git
├── .trae/                    # Trae IDE 项目规则目录，当前规则存放于此
├── docs/                     # 【核心约束】仅存放纯Markdown文档，禁止放任何配置文件
├── public/                   # 静态资源目录，仅存放logo、favicon、图片等静态文件
├── scripts/                  # 自定义脚本目录
├── site/                     # 构建产物目录，mkdocs build 自动生成，禁止手动修改
├── .gitignore                # Git忽略配置
├── README.md                 # 项目说明文档
└── mkdocs.yml                # 【唯一配置入口】项目核心配置文件，仅允许在根目录存放
```

## 二、各目录用途与约束规则
1.  **mkdocs.yml 核心配置文件**
    - 项目唯一的配置入口，必须放在项目根目录，禁止在docs等其他目录存放配置文件
    - 所有配置修改必须增量更新，禁止覆盖原有功能配置、容错规则
    - 必须保持 `strict: false` 关闭严格模式，禁止开启任何会导致构建中断的严格校验
    - 必须保持 `docs_dir: ./docs`、`site_dir: ./site`、`extra_dirs: [./public]` 目录配置不变

2.  **docs 目录（纯文档目录）**
    - 仅允许存放Markdown格式的文档文件，禁止存放任何配置文件、脚本文件、非文档类内容
    - 目录名称无强制限制，但必须符合文件系统合法性规范：
      - 禁止使用特殊字符：`/ \ : * ? " < > |`
      - 禁止以空格、点开头或结尾
      - 建议使用中文、英文、数字、下划线、短横线的组合
    - 【强制约束】目录层级最多到3级，禁止创建超过3级的子目录
      - 示例：`docs/一级目录/二级目录/三级目录/` 为最大深度
      - 禁止：`docs/一级/二级/三级/四级/`
    - 文档内的静态资源引用必须使用相对路径，指向public目录的资源
    - 支持原生HTML标签、标准Markdown语法，无Vue模板编译限制，无需处理语法容错

3.  **public 目录（静态资源目录）**
    - 仅存放favicon.svg、logo、图片、附件等静态资源文件

5.  **config 目录（项目配置目录）** 
    - 仅存放项目核心配置文件 `mkdocs.yml`，禁止存放任何其他配置文件
    - 禁止在该目录下创建子目录，禁止在该目录下存放任何文件

5.  **site 目录（构建产物目录）**
    - 执行 `mkdocs build` 命令后自动生成的纯静态文件目录
    - 禁止手动修改该目录内的任何文件，禁止提交该目录到Git仓库，必须加入.gitignore
    - 该目录内的所有文件可直接上传到阿里云OSS/腾讯云COS，开启静态网站托管即可访问
