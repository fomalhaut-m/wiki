Luke's Wiki MkDocs 项目 Trae 开发规则

一、项目目录结构强制规范（严格遵循现有结构，禁止修改）
project-root/
├── .github/workflows/       # GitHub Actions 自动化部署配置
├── .idea/                    # IDE 配置文件，禁止提交到Git
├── .trae/                    # Trae IDE 项目规则目录，当前规则存放于此
├── docs/                     # 【核心约束】仅存放纯Markdown文档，禁止放任何配置文件
├── public/                   # 静态资源目录，仅存放logo、favicon、图片等静态文件
├── scripts/                  # 自定义脚本目录
├── site/                     # 构建产物目录，mkdocs build 自动生成，禁止手动修改
├── .gitignore                # Git忽略配置
├── README.md                 # 项目说明文档
└── mkdocs.yml                # 【唯一配置入口】项目核心配置文件，仅允许在根目录存放

二、各目录用途与约束规则
1. mkdocs.yml 核心配置文件
- 项目唯一的配置入口，必须放在项目根目录，禁止在docs等其他目录存放配置文件
- 所有配置修改必须增量更新，禁止覆盖原有功能配置、容错规则
- 必须保持 `strict: false` 关闭严格模式，禁止开启任何会导致构建中断的严格校验
- 必须保持 `docs_dir: ./docs`、`site_dir: ./site`、`extra_dirs: [./public]` 目录配置不变

2. docs 目录（纯文档目录）
- 仅允许存放Markdown格式的文档文件，禁止存放任何配置文件、脚本文件、非文档类内容
- 目录名称需符合文件系统合法性规范：禁止使用特殊字符（`/ \ : * ? " < > |`），禁止以空格、点开头或结尾，建议使用中文、英文、数字、下划线、短横线的组合
- 【强制约束】目录层级最多到3级，禁止创建超过3级的子目录
  - 示例：`docs/一级目录/二级目录/三级目录/` 为最大深度
  - 禁止：`docs/一级/二级/三级/四级/`
- 文档内的静态资源引用必须使用相对路径，指向public目录的资源
- 支持原生HTML标签、标准Markdown语法，无Vue模板编译限制，无需处理语法容错

3. public 目录（静态资源目录）
- 仅存放favicon.svg、logo、图片、附件等静态资源文件

4. scripts 目录（自定义脚本目录）
- 存放项目相关自定义脚本，脚本命名需符合后续“文件名命名规范”，禁止存放无关脚本或非脚本文件

5. site 目录（构建产物目录）
- 执行 `mkdocs build` 命令后自动生成的纯静态文件目录
- 禁止手动修改该目录内的任何文件，禁止提交该目录到Git仓库，必须加入.gitignore
- 该目录内的所有文件可直接上传到阿里云OSS/腾讯云COS，开启静态网站托管即可访问

6. 其他目录补充
- .github/workflows/：仅存放GitHub Actions自动化部署配置文件，配置文件命名需符合“文件名命名规范”
- .idea/：IDE自动生成的配置文件，禁止提交到Git仓库，需在.gitignore中配置忽略
- .trae/：Trae IDE项目规则目录，仅存放当前开发规则文件，禁止存放其他无关内容

三、目录命名规范
1. 通用原则
- 简洁、语义明确，避免模糊、笼统命名（如：`笔记`、`新建文件夹`）
- 全目录统一语言风格：优先中文，技术类目录可用英文/缩写，禁止中英混合
- 长度控制：2–8个字，不超过12个字符
- 禁止特殊字符：`/ \ : * ? " < > |`，禁止首尾空格、点

2. 格式与分隔
- 中文目录：直接使用中文，不加空格、不加符号
  - 正确：`Java基础`、`项目复盘`、`读书笔记`
  - 错误：`Java 基础`、`项目-复盘`、`_读书笔记`
- 英文/技术目录：小写 + 短横线 `-` 分隔（kebab-case）
  - 正确：`spring-boot`、`design-pattern`
  - 错误：`SpringBoot`、`spring_boot`、`spring boot`
- 排序类目录：**尽量去掉文件名/目录名中的数字序号（123、一二三、01-、02- 等）**；若确需排序，必须与对应目录 `index.md` 中列出的顺序严格保持一致，**禁止文件系统顺序与页面目录顺序不一致**。
  - 推荐：`环境搭建`、`快速入门`（无序号，靠 index.md 控制顺序）
  - 不得已保留序号时：`01-环境搭建`、`02-快速入门`，且 index.md 列表顺序必须完全相同

3. 目录索引文档
- 每个目录必须有入口文档：`README.md`（大写）
- 作用：说明该目录用途、目录结构、使用注意事项

四、文件名命名规范
1. 通用原则
- 语义清晰：文件名即标题，无需打开即可知道内容
- 格式统一：一律小写（英文）/中文正常书写，Markdown文件后缀固定为 `.md`
- 禁止特殊字符：`/ \ : * ? " < > |`，禁止首尾空格、点
- **清理冗余序号**：文件名中若包含无意义数字/汉字序号（123、一二三、01-、02- 等），**优先删除**，保持文件名干净；若因内容逻辑必须保留序号，**必须和所在目录 index.md 中的展示顺序完全一致**，避免出现“文件顺序乱、页面顺序也乱”的情况。

2. Markdown 文档命名
（1）普通内容文档
- 中文：直接用标题，不加符号、不加数字（除非排序）
  - 正确：`Java 线程池原理.md`、`2026-05-08 周总结.md`
  - 错误：`java线程池原理.md`、`_Java线程池原理.md`
- 英文/技术：小写 + 短横线 `-`
  - 正确：`spring-boot-auto-config.md`
  - 错误：`SpringBootAutoConfig.md`、`spring_boot_auto_config.md`

（2）排序/序列文档
- 首选：**不加序号，仅在 index.md 中手动维护顺序列表**
- 不得已保留序号：`两位数字-标题.md`，且必须与 index.md 顺序一致
  - 正确：`01-环境搭建.md`、`02-基础配置.md`（index.md 中也按此顺序书写）
  - 错误：`1-环境搭建.md`、`2_基础配置.md`；或文件名有号、index.md 顺序不一致

（3）首页/入口文档
- 目录主页：`README.md`（大写，每个目录都要有）
- 站点首页：`index.md`（放在 `docs/` 根目录）

3. 静态资源文件（public/ 下）
- 图片：`[语义名].[后缀]`，多图可加序号
  - 正确：`logo.svg`、`01-架构图.png`、`avatar-luke.jpg`
- 附件：`[标题]-[日期].[后缀]`
  - 正确：`2026-05-08-需求文档.pdf`

4. 脚本文件（scripts/ 下）
- 格式：`[功能描述].[脚本后缀]`，英文使用小写+短横线分隔
  - 正确：`deploy-oss.sh`、`backup-docs.ps1`
  - 错误：`DeployOSS.sh`、`backup_docs.ps1`

五、强制禁止事项
- 禁止创建超过3级的子目录
- 禁止文件名/目录名中英混合、大小写混乱
- 禁止使用无意义命名：`新建文件夹`、`未命名`、`文档1`等
- 禁止在 `docs/` 下放配置文件、脚本、非md文件
- 禁止在 `public/` 下放源码、配置、临时文件
- 禁止手动修改 `site/` 目录下的任何文件，禁止提交该目录到Git
- 禁止修改 `mkdocs.yml` 中的核心目录配置（docs_dir、site_dir、extra_dirs）
- 禁止提交 `.idea/` 目录到Git仓库
- 禁止在文件名/目录名中随意使用数字/汉字序号；确需排序时，**必须与对应 index.md 列表顺序完全一致**。

六、特殊文件处理规则
1. 图片文件处理
- 对项目中的所有 `.png`、`.jpg`、`.gif`、`.svg` 等图片文件保持原样，暂不进行处理

2. 代码文件处理
- 对项目中的所有 `.sql` 和 `.java` 文件执行内容分析，识别文件间及与附近文档的关联关系，并基于分析结果自动编排生成或整合到 Markdown 文档中
- 确保内容组织有序、易于查阅且保持代码格式完整可读
- 代码示例必须使用正确的语法高亮标记（如 ```java、```sql）

3. 配置文件处理
- `.conf`、`.txt` 等配置文件应转换为 Markdown 文档
- 格式要求：标题 + 描述 + ``` 包裹原始内容
- 示例：
  ```markdown
  # 配置文件名

  > 配置文件说明

  ```conf
  [配置文件原始内容]
  ```
  ```
- 保留原配置文件在原位置不做删除
