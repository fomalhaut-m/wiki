# VitePress 知识库项目约束规则

## 一、项目概述

本规则适用于 `C:\work\wiki\docs\` 目录下的 VitePress 知识库项目。

---

## 二、目录结构规范

### 1. 标准结构

```
docs/
├─ index.md              # 站点首页
├─ [分类文件夹]/
│  ├─ index.md           # 分类入口页
│  ├─ [文档1].md         # 文档（1级）
│  ├─ [文档2].md
│  ├─ [子文件夹]/        # 子分类（2级）
│  │  ├─ index.md
│  │  ├─ *.md            # 子文档
│  │  └─ assets/         # 资源目录（3级）
│  └─ assets/            # 资源目录（2级）
└─ _归档/                # 归档目录
```

### 2. 目录层级限制（强制）

- **最大层级**：3 级目录结构
- **计算方式**：`docs` 目录下第1级 → 第2级 → 第3级
- **资源目录**：每个子目录下可创建 `assets/` 或 `images/` 存放资源（不计入层级限制）
- **禁止**：超过3级目录嵌套

### 3. 文件夹命名规范

- **强制**：仅使用中文、英文、数字、短横线「-」
- **禁止**：空格、特殊字符、emoji、中文标点、括号
- **示例**：`软件开发`、`产品经理`、`AI` ✅
- **错误**：`AI tool`、`产品经理(进阶)`、`学习📚笔记` ❌

### 4. 文件命名规范

- **强制**：仅使用 `.md` 后缀
- **强制**：仅使用中文、英文、数字、短横线「-」
- **禁止**：空格、特殊字符、括号
- **示例**：`Blender入门.md`、`Redis数据类型.md` ✅
- **错误**：`01.Blender 与 Ae.md`、`笔记 (1).md` ❌

## 三、文档编写规范

### 1. Frontmatter 规范

每个 `.md` 文件必须包含最简 Frontmatter：

```yaml
---
title: 文档标题
---
```

**禁止**：
- 添加多余字段（description、author、date 等）
- 使用复杂的 Frontmatter 配置

### 2. 标题层级规范

- **强制**：使用标准的 `#` 标题符号
- **禁止**：使用 `====` 或 `----` 作为标题分隔符
- **推荐**：保持 2-3 级标题层级，不要过深嵌套

```markdown
# 一级标题（文章标题）
## 二级标题（章节）
### 三级标题（子章节）
```

### 3. 代码块规范

- **强制**：使用 ``` 包裹代码
- **推荐**：指定语言以启用语法高亮

```markdown
```javascript
const hello = 'world';
```
```

### 4. 链接路径规范

- **强制**：使用相对路径
- **强制**：链接目标必须存在
- **推荐**：链接文本应简洁明了

```markdown
[本地文档](./relative-path.md)
[外部链接](https://example.com)
```

### 5. 图片引用规范

- **强制**：图片必须存放在 `assets` 目录或文档同级的 `images` 目录
- **强制**：使用正确的相对路径引用

```markdown
![图片描述](./assets/image.png)
```

## 四、允许与禁止清单

### 1. 允许的格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 主文档 | `.md` | 普通 Markdown 文件 |
| 组件 | `.vue` | Vue 单文件组件 |
| 图片 | jpg/png/svg/webp/gif | 静态图片资源 |
| 视频 | mp4/webm | 嵌入式视频 |
| 文档 | pdf | PDF 文档 |

### 2. 禁止的格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 增强文档 | `.mdx` | 禁止使用 |
| 富文本 | `.doc/.docx/.xls/.xlsx/.ppt/.pptx` | 禁止进入主文档 |
| 其他 | `.txt`、`.rtf` | 应转换为 md 格式 |

### 3. 允许的功能

- **代码块高亮** - 原生支持，无需配置
- **Mermaid 图表** - 使用 ```mermaid 语法
- **自定义提示容器** - 使用 `::: tip / warning / note`
- **Vue 组件** - 可在 md 中直接使用 Vue 标签

### 4. 禁止的功能

- **数学公式** - 不启用
- **复杂 Markdown 扩展** - 不使用
- **第三方格式插件** - 不接入（思维导图、Office导入等）
- **复杂主题定制** - 仅使用默认布局

## 五、文件夹结构要求

### 1. 入口文件要求

- **强制**：每个文件夹必须有 `index.md` 作为入口
- **强制**：`index.md` 内容应包含该分类的文档导航
- **推荐**：`index.md` 只包含导航和简要说明，不写长篇内容

### 2. 层级限制

- **最大层级**：3 级目录（含docs根目录）
- **解释**：docs → 分类 → 子分类 → 子子分类
- **资源目录**：`assets/` 和 `images/` 目录不计入层级限制
- **禁止**：超过3级嵌套

### 3. 资源目录

- **位置**：`assets/` 目录与 `.md` 文件同级
- **用途**：存放该文档/分类直接引用的静态资源
- **禁止**：将 `assets` 用于存储非引用资源

## 六、内容管理规范

### 1. 文档合并原则

当多个文档主题高度相关时，可以合并为一个文档：

**允许合并的情况**：
- 主题完全相同或高度重叠
- 内容存在重复
- 逻辑上可以整合

**合并要求**：
- 保留所有有价值内容
- 保持逻辑连贯性
- 更新相关链接引用

### 2. 文档拆分原则

当单个文档内容庞杂时，可以拆分为多个文档：

**允许拆分的情况**：
- 内容涵盖多个独立主题
- 篇幅过长
- 不同读者群体需要

**拆分要求**：
- 保持内容完整性
- 添加适当的交叉引用
- 更新父级目录导航

### 3. 内容整理原则

**允许的整理**：
- 修正错别字和语法错误
- 优化段落结构和排版
- 规范化列表和代码块格式
- 删除无意义的重复内容

**禁止的整理**：
- 删除用户原创的个人见解和经验总结
- 修改核心知识点和原始数据
- 改变原文的核心逻辑

## 七、资源引用规范

### 1. 本地资源

```markdown
<!-- 图片引用 -->
![描述](./assets/image.png)

<!-- 文件引用 -->
[下载文件](./assets/document.pdf)
```

### 2. 资源缺失处理

当引用的本地资源不存在时：
1. **保留**原文引用代码和文本
2. **添加**注释标记：`<!-- [资源缺失] 引用文件不存在 -->`
3. **禁止**删除原文引用

### 3. 外部链接

- **推荐**：使用 HTTPS 链接
- **推荐**：链接文本应描述链接目标
- **处理**：失效链接保留原样，不删除

---

## 八、异常处理规则

| 情况 | 处理方式 |
|------|----------|
| 空文件/0字节文件 | 删除或移至归档目录 |
| 损坏无法读取 | 移至 `_归档/损坏文件/` 目录 |
| 路径包含空格 | 将空格替换为 `-` |
| 重复文件 | 保留最新修改的，删除其他 |
| 权限不足 | 记录日志，跳过该文件 |

## 十、深层目录整理指南

### 超过3级的目录处理方法

当发现目录超过3级时，可采用以下方法整理：

1. **提升层级**：将深层子目录的文件上移至第3级
2. **合并文档**：将深层目录的多个文档合并为一个
3. **删除中间层级**：删除无实质内容的中间目录
4. **资源外置**：将深层 assets 目录的文件移至上层

### 示例

```
# 整理前（4级）
docs/Java/框架/Spring-Boot/SpringBoot-进阶/深入原理/进阶篇/源码分析/核心原理.md

# 整理后（3级）
docs/Java/框架/SpringBoot深入原理.md
```
